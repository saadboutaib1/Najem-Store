# Security Policy

## Reporting a vulnerability

Please report suspected security issues privately to the project owner or store administrator through the established private project contact channel. Do not open public issues with exploit details, secrets, customer data, or screenshots containing private information.

Include:

- A clear description of the issue.
- Affected route, page, or endpoint.
- Steps to reproduce in a safe test environment.
- Expected impact.
- Any suggested fix, if known.

## Handling

Security reports should be reviewed before public disclosure. Confirmed issues should be fixed, tested, and deployed before details are shared broadly.

## Scope

This policy covers the MAGHRIB OUD frontend, backend API, admin dashboard, PWA assets, product/order flows, and deployment configuration files in this repository.
